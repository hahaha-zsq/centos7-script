Hadoop 3.3.6 HDFS + YARN 集群部署

02_deploy_hadoop.sh

自动化部署 Hadoop HDFS 分布式文件系统和 YARN 资源管理器集群，支持在线下载和离线安装两种模式，自动完成配置生成、分发、格式化和启动。

3 节点集群部署方案

以下是 3 台服务器组成 Hadoop 集群时各节点的角色分配和服务分布：

MASTER

node1

NameNode
HDFS 元数据管理，维护文件目录树和数据块映射
ResourceManager
YARN 资源调度中心，分配容器和管理集群资源
DataNode
同时承担数据存储，增加可用存储空间和数据本地性
NodeManager
同时承担容器管理，增加集群可用计算资源
ZooKeeper
分布式协调服务，为其他组件提供一致性支持
Kafka Broker
消息队列代理，处理实时数据流
WORKER

node2

DataNode
HDFS 数据存储节点，管理本地磁盘上的数据块
NodeManager
YARN 工作节点代理，管理本节点的容器生命周期
Secondary NameNode
定期合并 NameNode 的编辑日志和镜像文件
ZooKeeper
分布式协调服务副本
Kafka Broker
消息队列代理副本
WORKER

node3

DataNode
HDFS 数据存储节点，管理本地磁盘上的数据块
NodeManager
YARN 工作节点代理，管理本节点的容器生命周期
ZooKeeper
分布式协调服务副本，形成 3 节点仲裁
Kafka Broker
消息队列代理副本
服务所在节点端口作用
NameNodenode19870 (Web) / 9000 (RPC)HDFS 主节点，管理元数据，客户端通过它读写文件
ResourceManagernode18088 (Web)YARN 主节点，接收作业请求并分配资源给各 NodeManager
DataNodenode1, node2, node39864存储实际数据块，定期向 NameNode 汇报块状态
NodeManagernode1, node2, node38042管理容器（Container）的启动、监控和资源使用
Secondary NameNodenode29868 (Web)定期 checkpoint，合并 fsimage + edits，防止单点故障恢复太慢
ZooKeepernode1, node2, node321813 节点选举仲裁，提供分布式锁和配置管理
Kafka Brokernode1, node2, node39092分布式消息队列，Flink 实时消费的数据源
ⓘ
HDFS 副本数 = 2 — 3 节点集群中所有节点均运行 DataNode（共 3 个），HDFS 副本数默认设为 2。若需更高容错能力，可设为 3（每个数据块在全部 3 个 DataNode 上各存一份）。Kafka 和 ZooKeeper 同样部署在全部 3 个节点以保证高可用。

HDFS + YARN 架构与通信方式

3 节点 Hadoop 集群的完整架构，包含 HDFS 文件系统、YARN 资源管理和各组件之间的通信协议与端口。

通信方向协议端口数据内容频率
DataNode → NameNodeHeartbeat9000 (RPC)节点存活信号，包含磁盘使用率、剩余空间默认 3 秒
DataNode → NameNodeBlock Report9000 (RPC)该节点上所有数据块的列表和校验和默认 6 小时
NameNode → DataNodeRPC 指令9000复制/删除/移动数据块的指令按需
DataNode ↔ DataNodeDataStream9866 (Pipeline)数据块副本的管道传输（写入时串行复制）写入时
NameNode → SecondaryNNHTTP9868fsimage + edits 文件，用于 checkpoint 合并默认 1 小时
ⓘ
NameNode 如何知道数据存放在哪些 DataNode？
DataNode 启动后会向 NameNode 注册，之后定期发送 Heartbeat（证明存活）和 Block Report（汇报持有的数据块）。NameNode 在内存中维护块映射表，不需要持久化存储块位置信息。
通信方向协议端口数据内容频率
NodeManager → ResourceManagerHeartbeat8030 (RM Scheduler)节点资源使用情况（CPU/内存已用/剩余）、运行中的容器状态默认 1 秒
ResourceManager → NodeManagerRPC 指令8042启动/杀死容器的指令，包含容器资源分配按需
Container → NodeManager本地通信本地容器状态上报、日志本地写入运行中
ApplicationMaster → ResourceManagerRPC8030申请/释放资源，汇报任务进度按需
ApplicationMaster → NodeManagerRPC8042管理容器生命周期（启动、停止、查询状态）按需
ⓘ
YARN 作业执行流程：
1. Client 提交作业到 ResourceManager
2. RM 在某个 NodeManager 上启动 ApplicationMaster
3. AM 向 RM 申请资源（Container）
4. RM 分配 Container 到各 NodeManager
5. AM 在 Container 中启动任务（Map/Reduce）
6. 任务完成后 AM 向 RM 注销

HDFS 写入流程 (Pipeline)

1
Client → NameNode :9000
请求创建文件，NameNode 检查权限并记录元数据，返回一组 DataNode 列表
2
Client → DN1 → DN2 → DN3 (Pipeline)
Client 建立到第一个 DataNode 的管道，数据以 Packet 为单位沿管道串行传输，每个 DN 确认收到后传给下一个
3
ACK 沿管道反向返回
最后一个 DN 确认后，ACK 沿管道反向传回 Client，表示数据已持久化到所有副本
4
Client → NameNode :9000
文件写完后，Client 通知 NameNode 关闭文件，NameNode 提交元数据

HDFS 读取流程

1
Client → NameNode :9000
请求读取文件，NameNode 返回文件的块列表，每个块对应的 DataNode 地址（按网络拓扑排序）
2
Client → DataNode :9864
Client 选择最近的 DataNode（优先本地 > 同机架 > 其他机架），直接从 DN 读取数据块
3
失败重试
如果某个 DN 读取失败，Client 自动切换到该块的另一个副本所在的 DN 继续读取

Hadoop 启动与管理脚本

Hadoop 提供了一套完整的脚本来管理集群服务的启停，位于 $HADOOP_HOME/sbin/ 和 $HADOOP_HOME/bin/ 目录下。

脚本作用启动的进程执行位置
start-dfs.sh 启动整个 HDFS 集群 NameNode + SecondaryNameNode + 所有 DataNode Master 节点执行，SSH 到各 Worker
stop-dfs.sh 停止整个 HDFS 集群 同上，全部关闭 Master 节点执行
start-yarn.sh 启动整个 YARN 集群 ResourceManager + 所有 NodeManager Master 节点执行，SSH 到各 Worker
stop-yarn.sh 停止整个 YARN 集群 同上，全部关闭 Master 节点执行
start-all.sh 启动 HDFS + YARN（等同于依次执行上面两个） 全部守护进程 Master 节点执行
stop-all.sh 停止 HDFS + YARN 全部守护进程 Master 节点执行
hadoop-daemon.sh 在本机启动/停止单个守护进程 指定的单个进程 任意节点执行
hadoop-daemons.sh 在所有 Worker 上启动/停止守护进程 各 Worker 上的指定进程 Master 节点执行
yarn-daemon.sh 在本机启动/停止 YARN 守护进程 RM 或 NM 单个进程 任意节点执行
yarn-daemons.sh 在所有 Worker 上启动/停止 YARN 守护进程 各 Worker 上的 NM Master 节点执行
mr-jobhistory-daemon.sh 启动/停止 MapReduce JobHistory Server JobHistoryServer Master 节点执行
命令子命令作用示例
hdfs dfs -ls 列出目录内容 hdfs dfs -ls /
-put 上传本地文件到 HDFS hdfs dfs -put local.txt /data/
-get 从 HDFS 下载文件到本地 hdfs dfs -get /data/file.txt ./
hdfs dfsadmin -report 查看集群状态报告 hdfs dfsadmin -report
-safemode 管理安全模式 hdfs dfsadmin -safemode leave
-refreshNodes 刷新节点列表（扩容/缩容后） hdfs dfsadmin -refreshNodes
yarn application -list 列出运行中的应用 yarn application -list
node -list 列出所有 NodeManager yarn node -list
mapred job -list 列出 MapReduce 作业 mapred job -list
--daemon start historyserver 启动 JobHistory Server mapred --daemon start historyserver

常见操作场景

首次启动集群

完整流程 
# 1. 格式化 NameNode（仅首次）
hdfs namenode -format -force -nonInteractive

# 2. 启动 HDFS
start-dfs.sh

# 3. 启动 YARN
start-yarn.sh

# 4. 启动 JobHistory Server
mapred --daemon start historyserver

# 5. 验证
jps                              # 查看 Java 进程
hdfs dfsadmin -report            # 查看 HDFS 状态
yarn node -list                  # 查看 YARN 节点

日常重启集群

标准流程 
# 停止（先停 YARN 再停 HDFS，顺序很重要）
stop-yarn.sh
stop-dfs.sh

# 启动（先启 HDFS 再启 YARN）
start-dfs.sh
start-yarn.sh
mapred --daemon start historyserver

重启单个服务

精准操作 
# 在 node1 上重启 NameNode
hdfs --daemon stop namenode
hdfs --daemon start namenode

# 在 node2 上重启 DataNode
hdfs --daemon stop datanode
hdfs --daemon start datanode

# 在 node1 上重启 ResourceManager
yarn --daemon stop resourcemanager
yarn --daemon start resourcemanager

# 在 node2 上重启 NodeManager
yarn --daemon stop nodemanager
yarn --daemon start nodemanager
⚠
启动顺序很重要：
• 启动：先 HDFS → 再 YARN → 最后 JobHistory
• 停止：先 YARN → 再 HDFS → 最后 JobHistory
原因：YARN 依赖 HDFS 存储，必须在 HDFS 就绪后才能启动。停止时反过来，先停上层应用再停底层存储。

配置文件详解

脚本自动生成以下 6 个配置文件并分发到所有节点的 $HADOOP_HOME/etc/hadoop/ 目录。

core-site.xml

Hadoop 核心配置 
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://node1:9000</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/data/bigdata/tmp</value>
    </property>
    <property>
        <name>io.file.buffer.size</name>
        <value>131072</value>
    </property>
</configuration>
fs.defaultFS
默认文件系统 URI
指定 HDFS 的 NameNode 地址，所有客户端通过此地址访问 HDFS。格式: hdfs://主机名:RPC端口
hadoop.tmp.dir
Hadoop 临时目录
NameNode、DataNode 等进程的临时数据存放位置。需确保目录存在且 Hadoop 用户有读写权限。
io.file.buffer.size
IO 缓冲区大小 (字节)
读写文件时的缓冲区大小，默认 4096，设为 128KB (131072) 可显著提升大文件读写性能。

hdfs-site.xml

HDFS 分布式文件系统配置 
<configuration>
    <property>
        <name>dfs.replication</name>
        <value>2</value>
    </property>
    <property>
        <name>dfs.namenode.name.dir</name>
        <value>file:///data/bigdata/hdfs/name</value>
    </property>
    <property>
        <name>dfs.datanode.data.dir</name>
        <value>file:///data/bigdata/hdfs/data</value>
    </property>
    <property>
        <name>dfs.namenode.http-address</name>
        <value>node1:9870</value>
    </property>
    <property>
        <name>dfs.namenode.secondary.http-address</name>
        <value>node3:9868</value>
    </property>
    <property>
        <name>dfs.permissions</name>
        <value>false</value>
    </property>
</configuration>
dfs.replication
HDFS 副本数
每个数据块的副本数量。3 节点集群建议设为 2，5 节点以上可设为 3。副本数不能超过 DataNode 数量。
dfs.namenode.name.dir
NameNode 元数据目录
存放 HDFS 文件系统元数据（文件目录树、块映射等）。此目录数据极其重要，丢失将导致整个 HDFS 不可用。建议配置多个目录做冗余。
dfs.datanode.data.dir
DataNode 数据目录
存放实际的数据块文件。可配置多个逗号分隔的目录，实现跨磁盘存储以提升 IO 性能。
dfs.namenode.http-address
NameNode Web UI 地址
HDFS 管理界面的 HTTP 地址，可通过浏览器访问查看集群状态、浏览文件系统。
dfs.namenode.secondary.http-address
SecondaryNameNode 地址
SecondaryNameNode 定期合并 NameNode 的编辑日志和镜像文件，减轻 NameNode 启动时的合并负担。应部署在与 NameNode 不同的节点上。
dfs.permissions
HDFS 权限检查
设为 false 禁用文件权限检查，方便开发测试。生产环境建议设为 true 并配置正确的用户权限。

yarn-site.xml

YARN 资源管理器配置 
<configuration>
    <property>
        <name>yarn.resourcemanager.hostname</name>
        <value>node1</value>
    </property>
    <property>
        <name>yarn.resourcemanager.webapp.address</name>
        <value>node1:8088</value>
    </property>
    <property>
        <name>yarn.nodemanager.resource.memory-mb</name>
        <value>8192</value>
    </property>
    <property>
        <name>yarn.nodemanager.resource.cpu-vcores</name>
        <value>4</value>
    </property>
    ... (更多配置见下方)
yarn.resourcemanager.hostname
ResourceManager 主机名
YARN 的中央资源调度器所在节点，负责管理所有 NodeManager 的资源分配。
yarn.nodemanager.resource.memory-mb
NodeManager 可分配内存 (MB)
每个 Worker 节点上可供 YARN 容器使用的总内存。应小于节点实际可用内存，预留部分给操作系统和其他进程。
yarn.nodemanager.resource.cpu-vcores
NodeManager 可分配 CPU 核数
每个 Worker 节点上可供 YARN 容器使用的总 CPU 核心数。
yarn.scheduler.minimum-allocation-mb
单容器最小内存
YARN 分配给每个容器的最小内存，低于此值不会分配。
yarn.scheduler.maximum-allocation-mb
单容器最大内存
单个容器可申请的最大内存，不能超过 NodeManager 总内存。
yarn.nodemanager.aux-services
辅助服务
设为 mapreduce_shuffle，让 NodeManager 支持 MapReduce 的 Shuffle 操作（中间数据排序和传输）。
yarn.log-aggregation-enable
日志聚合
启用后，应用完成后日志会从各 NodeManager 聚合到 HDFS，方便统一查看。
yarn.nodemanager.env-whitelist
环境变量白名单
容器中可继承的环境变量列表，确保 JAVA_HOME 等关键变量传递到 YARN 容器中。

mapreduce.framework.name
MapReduce 运行框架
设为 yarn 表示 MapReduce 任务运行在 YARN 上，由 YARN 负责资源分配。也可设为 local 用于本地调试。
mapreduce.jobhistory.address
JobHistory Server RPC 地址
MapReduce 任务历史记录服务的 RPC 端口，客户端通过此地址获取已完成任务的信息。
mapreduce.jobhistory.webapp.address
JobHistory Server Web 地址
通过浏览器访问此地址查看 MapReduce 任务的历史执行详情、计数器、日志等。
yarn.app.mapreduce.am.env
ApplicationMaster 环境变量
设置 MapReduce ApplicationMaster 的 HADOOP_MAPRED_HOME 环境变量，确保能找到依赖的 JAR 包。map.env 和 reduce.env 同理。

JAVA_HOME
JDK 安装目录
Hadoop 必须运行在 JDK 环境上，此处指定 JDK 的根目录。
HADOOP_LOG_DIR
日志目录
所有 Hadoop 进程的日志输出目录，包括 NameNode、DataNode、ResourceManager、NodeManager 的运行日志。
HDFS_*_USER / YARN_*_USER
进程运行用户
指定各守护进程以哪个用户身份运行。生产环境不建议用 root，应使用专用用户如 hadoop。
HADOOP_NAMENODE_OPTS
NameNode JVM 参数
NameNode 管理整个 HDFS 的元数据，内存需求最大。元数据量大时需要增加 -Xmx 值。
HADOOP_DATANODE_OPTS
DataNode JVM 参数
DataNode 主要负责数据块存储，内存需求相对较小。
