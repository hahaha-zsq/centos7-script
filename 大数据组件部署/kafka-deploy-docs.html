Kafka 3.x 集群部署 自动化脚本

03_deploy_kafka.sh

自动化部署 Kafka 集群（包含 ZooKeeper），支持在线下载和离线安装两种模式，自动完成配置生成、分发、启动和验证。

脚本功能概述

◇ 核心功能

✓ 在线 / 离线双模式部署
✓ 自动生成 ZooKeeper 和 Kafka 配置
✓ SSH 远程分发到所有节点
✓ 服务状态检查后启动
✓ 部署后自动验证连通性
✓ 试运行预览模式
✓ 详细日志和错误处理
✓ 资源自动清理机制

使用方法

bash
# 在线部署（默认）
./03_deploy_kafka.sh

# 离线部署
./03_deploy_kafka.sh -m offline

# 试运行查看执行计划
./03_deploy_kafka.sh -d

# 启用详细输出调试
./03_deploy_kafka.sh -v

参数说明

短参数长参数说明默认值
-m--mode安装模式: online 或 offlineonline
-d--dry-run试运行模式（仅显示将要执行的操作）false
-v--verbose启用详细输出false
-h--help显示帮助信息—

重要注意事项

ⓘ
离线模式注意：在离线模式下，请确保将 ZooKeeper 和 Kafka 的安装包放置在 ${OFFLINE_DIR}/ 目录中，文件名需与脚本中定义的 ZOOKEEPER_PACKAGE 和 KAFKA_PACKAGE 变量匹配。
ⓘ
前置条件：请确保目标机器已配置好 SSH 免密登录，并且有 root 或具有 sudo 权限的用户可用。脚本会自动在目标机器上创建必要的目录和用户组。

集群架构说明

脚本支持灵活的集群拓扑，通过环境变量配置 ZooKeeper 节点和 Kafka Broker 节点列表。

组件作用端口备注
ZooKeeper分布式协调服务2181保存 Kafka 集群元数据，选举控制器
Kafka Broker消息代理节点9092处理生产和消费请求，存储消息
ⓘ
脚本会根据 ZK_HOSTS 和 KAFKA_BROKERS 环境变量自动分配 ZooKeeper 的 myid 和 Kafka 的 broker.id，无需手动配置。

端口说明和防火墙配置

在云服务器或防火墙环境中，需要开放以下端口以确保 Kafka 集群正常通信：

服务端口协议说明
ZooKeeper 客户端2181TCPKafka Broker 和客户端连接 ZooKeeper 的端口
ZooKeeper 节点间通信2888TCPZooKeeper 集群内部节点通信端口（Follower → Leader）
ZooKeeper 选举通信3888TCPZooKeeper 集群选举过程中的通信端口
Kafka Broker9092TCP生产者和消费者连接 Kafka 的端口
SSH22TCP脚本执行所需的远程登录端口（如需修改请参考下文）
ⓘ
重要提示：实际使用的端口可能受 env.sh 文件中的配置影响。如果在部署过程中发现端口被占用，请参考以下步骤修改端口配置：
bash
# 1. 编辑环境配置文件
vim env.sh

# 2. 找到端口配置部分（大约在第 132-145 行）：
# ==================== 端口配置 ====================
# Hadoop
HDFS_NAMENODE_PORT=9870
HDFS_NAMENODE_RPC_PORT=9000
YARN_RESOURCEMANAGER_PORT=8088
YARN_RESOURCEMANAGER_SCHEDULER_PORT=8030
YARN_NODEMANAGER_PORT=8042

# Kafka
KAFKA_PORT=9092  ← 修改此值

# Zookeeper
ZK_CLIENT_PORT=2181  ← 修改此值
ZK_PEER_PORT=2888    ← 修改此值
ZK_ELECTION_PORT=3888← 修改此值

# 3. 修改为未被占用的端口（例如）：
# Kafka
KAFKA_PORT=9093

# Zookeeper
ZK_CLIENT_PORT=2182
ZK_PEER_PORT=2889
ZK_ELECTION_PORT=3889

# 4. 保存文件并重新执行部署脚本
./03_deploy_kafka.sh
ⓘ
端口选择建议：选择 1024-65535 范围内的端口，避免与系统服务和其他常见应用冲突。修改后请确保防火墙/安全组规则也相应更新以放行新端口。
