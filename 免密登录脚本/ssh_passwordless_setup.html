什么是 SSH 免密登录？

◇ 工作原理

SSH 免密登录基于公钥认证机制，用户在本地生成密钥对（公钥+私钥），将公钥添加到目标服务器的 ~/.ssh/authorized_keys 文件中，之后连接时无需输入密码即可自动完成身份验证。

本地机器
私钥 (~/.ssh/id_rsa)
➜
目标服务器
公钥 (~/.ssh/authorized_keys)

使用方法

bash
# 1. 上传脚本到服务器
# 2. 添加执行权限
chmod +x ssh_passwordless_setup.sh

# 3. 运行脚本
./ssh_passwordless_setup.sh

# 4. 按提示输入目标服务器列表
# 格式: user@ip user2@ip2
# 示例: root@10.16.192.158 root@10.16.192.159

# 5. 根据提示输入每个服务器的SSH密码

常见问题与解决

⚠
问题：密钥已存在但仍需密码
原因：目标服务器 PubkeyAuthentication no 禁用了公钥认证。脚本会自动检测并修复，修复后仍失败请手动执行： 
ssh root@目标IP "sed -i 's/^PubkeyAuthentication no/PubkeyAuthentication yes/' /etc/ssh/sshd_config && systemctl restart sshd"
⚠
问题：Permission denied (publickey,gssapi-keyex,gssapi-with-mic,password)
原因：公钥未正确分发或权限不正确。请检查：
  • 目标服务器 ~/.ssh/ 目录权限是否为 700
  • authorized_keys 文件权限是否为 600
  • 公钥是否已添加到 authorized_keys 文件中
⚠
问题：单向免密vs双向免密
说明：SSH 免密登录是单向的。在 A 服务器运行脚本将公钥分发到 B，意味着 A 可以免密登录 B，但 B 不能免密登录 A。如果需要双向免密，需要在 B 服务器上也运行脚本。
⚠
问题：ssh-copy-id 命令不存在
解决：安装 openssh-clients 
# CentOS/RHEL
yum install -y openssh-clients

# Ubuntu/Debian
apt-get install -y openssh-client

手动配置命令参考

如果不想使用脚本，也可以手动执行以下命令完成配置：

bash
# 1. 生成密钥（如果不存在）
ssh-keygen -t rsa -b 4096 -C "注释" -f ~/.ssh/id_rsa -N ""

# 2. 分发公钥到目标服务器
ssh-copy-id -i ~/.ssh/id_rsa.pub root@目标IP

# 3. 手动修复权限（如果需要）
ssh root@目标IP "chmod 700 ~/.ssh && chmod 600 ~/.ssh/authorized_keys"

# 4. 启用公钥认证（如果需要）
ssh root@目标IP "sed -i 's/^#*PubkeyAuthentication no/PubkeyAuthentication yes/' /etc/ssh/sshd_config"
ssh root@目标IP "systemctl restart sshd"

# 5. 测试免密登录
ssh root@目标IP "hostname"

安全注意事项

⚠
私钥安全：妥善保管 ~/.ssh/id_rsa 私钥文件，不要泄露给他人。私钥相当于你的身份凭证，拥有私钥即可免密登录所有已配置的目标服务器。
⚠
密钥覆盖风险：运行脚本时，如果本地已存在密钥，脚本会提示是否重新生成。建议选择"否"以保留原有密钥，避免影响其他已配置的免密登录。
ⓘ
密钥复用：脚本会检查本地是否存在已有密钥，存在则复用，不会重复生成。这确保了同一把私钥可以登录多台服务器。
