# SSH免密认证部署脚本

## 概述

本脚本用于在多台服务器之间批量建立免密SSH认证，简化运维工作。脚本遵循防御性编程最佳实践，具备完善的错误处理、日志记录和验证机制。

## 功能特性

- 批量部署SSH公钥到多台服务器
- 自动生成RSA 4096位密钥对
- 自动检测已认证服务器，幂等设计
- 支持交叉认证（服务器之间互相免密）
- 支持强制重新生成密钥
- 试运行模式，预览变更
- 详细日志输出
- 部署后自动验证
- 安全的临时文件处理

## 使用方法

### 1. 创建服务器列表文件

创建 `servers.txt`，每行包含一台服务器的信息：

```
ip 用户名 密码
```

示例：

```
# 内网服务器
192.168.1.10 root MyPass123
192.168.1.11 root MyPass456
192.168.1.12 deploy MyPass789

# 跳板机
10.0.0.1 admin JumpPass999
```

支持的格式：
- 空行会被忽略
- 以 `#` 开头的行作为注释
- 每行三个字段：`IP` `用户名` `密码`，以空格分隔

### 2. 运行脚本

```bash
# 基本用法（单向认证：控制机到服务器）
./ssh_key_auth.sh -f servers.txt

# 交叉认证（服务器之间互相免密）
./ssh_key_auth.sh -f servers.txt -c

# 强制重新生成密钥
./ssh_key_auth.sh -f servers.txt --force

# 试运行模式（不实际执行）
./ssh_key_auth.sh -f servers.txt --dry-run

# 启用详细输出
./ssh_key_auth.sh -f servers.txt -v
```

### 3. 验证结果

脚本执行完成后会自动验证认证结果。也可以手动验证：

```bash
ssh root@192.168.1.10 'echo "认证成功"'
```

## 参数说明

| 参数 | 长参数 | 说明 | 默认值 |
|------|--------|------|--------|
| `-f` | `--file` | 服务器列表文件路径 | 必填 |
| `-k` | `--key` | SSH私钥路径 | `~/.ssh/id_rsa` |
| `-p` | `--port` | SSH端口 | `22` |
| `-t` | `--timeout` | 连接超时时间（秒） | `10` |
| `-o` | `--output` | 输出目录 | `./ssh_auth_output` |
| `-c` | `--cross` | 交叉认证模式（服务器之间互相免密） | `false` |
| | `--force` | 强制重新生成密钥 | `false` |
| `-v` | `--verbose` | 启用详细输出 | `false` |
| `-d` | `--dry-run` | 试运行模式 | `false` |
| `-h` | `--help` | 显示帮助信息 | - |

## 错误处理

### 常见错误及解决方法

| 错误信息 | 原因 | 解决方法 |
|----------|------|----------|
| `缺少必要命令: sshpass` | 未安装sshpass | `brew install sshpass` |
| `服务器文件不存在` | 文件路径错误 | 检查 `-f` 参数 |
| `第X行格式错误` | 服务器列表格式不正确 | 每行应为 `ip 用户名 密码` |
| `为 xxx@xxx 配置认证失败` | 密码错误或服务器不可达 | 检查密码和网络连通性 |
| `验证失败` | 公钥未正确部署 | 检查服务器sshd配置 |

### 调试模式

```bash
DEBUG=1 ./ssh_key_auth.sh -f servers.txt
```

## 安全注意事项

1. **密码安全**：服务器列表文件包含明文密码，使用后应及时删除
2. **文件权限**：建议设置 `chmod 600 servers.txt`
3. **密钥保护**：生成的私钥应妥善保管，不要泄露
4. **网络安全**：建议在内网环境使用，避免通过公网传输密码

```bash
# 设置服务器文件权限
chmod 600 servers.txt

# 使用后删除
rm -f servers.txt
```

## 高级用法

### 批量执行远程命令

部署完成后，可批量执行命令：

```bash
# 读取服务器列表并执行命令
while read -r ip user password; do
    ssh -o StrictHostKeyChecking=no "$user@$ip" "hostname && uptime"
done < servers.txt
```

### 使用环境变量

```bash
export SSH_KEY_PATH=~/.ssh/custom_key
export SSH_PORT=2222
export SSH_TIMEOUT=30

./ssh_key_auth.sh -f servers.txt
```

### 组合使用

```bash
# 交叉认证 + 强制重新生成 + 详细输出
./ssh_key_auth.sh -f servers.txt -c --force -v

# 交叉认证 + 试运行
./ssh_key_auth.sh -f servers.txt -c --dry-run
```

## 脚本设计原则

本脚本遵循以下防御性编程原则：

- `set -Eeuo pipefail` 严格模式
- 所有变量使用双引号
- ERR trap 错误捕获
- EXIT trap 清理临时文件
- 幂等设计，重复运行安全
- 结构化日志输出
- 输入验证和依赖检查
