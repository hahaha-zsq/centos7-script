Bash Script

SSH 免密认证部署脚本

在多台服务器之间批量建立免密 SSH 认证，遵循防御性编程最佳实践，具备完善的错误处理、日志记录和验证机制。

◇ 功能特性

✓ 批量部署 SSH 公钥到多台服务器
✓ 自动生成 RSA 4096 位密钥对
✓ 自动检测已认证服务器，幂等设计
✓ 支持交叉认证（服务器之间互相免密）
✓ 支持强制重新生成密钥
✓ 试运行模式，预览变更
✓ 详细日志输出与自动验证
✓ 安全的临时文件处理

快速开始

1

创建服务器列表文件

创建 servers.txt，每行包含 IP 用户名 密码：

servers.txt
# 内网服务器
192.168.1.10 root MyPass123
192.168.1.11 root MyPass456
192.168.1.12 deploy MyPass789

# 跳板机
10.0.0.1 admin JumpPass999
2

运行脚本

bash
# 单向认证（控制机到各服务器）
./ssh_key_auth.sh -f servers.txt

# 交叉认证（服务器之间互相免密）
./ssh_key_auth.sh -f servers.txt -c
3

验证结果

bash
ssh root@192.168.1.10 'echo "认证成功"'

参数说明

短参数 长参数 说明 默认值
-f --file 服务器列表文件路径 必填
-k --key SSH 私钥路径 ~/.ssh/id_rsa
-p --port SSH 端口 22
-t --timeout 连接超时时间（秒） 10
-o --output 密钥 / 日志输出目录 ./ssh_auth_output
-c --cross 交叉认证模式（服务器之间互相免密） false
--force 强制重新生成密钥 false
-v --verbose 启用详细输出 false
-d --dry-run 试运行模式，不实际执行 false
-h --help 显示帮助信息 —

常见问题

错误信息 原因 解决方法
缺少必要命令: sshpass 未安装 sshpass brew install sshpass
服务器文件不存在 文件路径错误 检查 -f 参数
第 X 行格式错误 服务器列表格式不正确 每行应为 ip 用户名 密码
配置认证失败 密码错误或服务器不可达 检查密码和网络连通性
验证失败 公钥未正确部署 检查服务器 sshd 配置
bash
# 启用调试模式
DEBUG=1 ./ssh_key_auth.sh -f servers.txt

安全注意事项

⚠
密码安全
服务器列表文件包含明文密码，使用后应及时删除。
chmod 600 servers.txt  |  rm -f servers.txt
⚠
网络安全
建议在内网环境使用，避免通过公网传输密码。生成的私钥应妥善保管。

工作流程

单向认证模式

默认模式，控制机到各服务器

检查依赖工具
↓
验证服务器文件
↓
生成 SSH 密钥
↓
遍历服务器列表
↓
检查是否已认证 → 复制公钥
↓
验证所有服务器
↓
输出部署结果

交叉认证模式

-c 参数，服务器之间互相免密

阶段 1: 控制机 → 服务器
(同单向认证)
↓
阶段 2: 服务器之间
↓
获取源服务器公钥
↓
复制公钥到目标服务器
↓
验证交叉认证结果

高级用法

批量执行远程命令

部署完成后，可批量执行命令

bash
while IFS= read -r line; do
    read -r ip user password <<< "$line"
    ssh -o StrictHostKeyChecking=no "$user@$ip" "hostname && uptime"
done < servers.txt

环境变量配置

通过环境变量覆盖默认参数

bash
export SSH_KEY_PATH=~/.ssh/custom_key
export SSH_PORT=2222
export SSH_TIMEOUT=30

./ssh_key_auth.sh -f servers.txt
